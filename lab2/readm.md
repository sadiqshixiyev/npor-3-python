## Описание задания
Дан массив целых чисел `nums` и целочисленное значение переменной `target`. Необходимо вернуть индексы двух чисел таким образом, чтобы они в сумме давали `target`.

### Ограничения и условия:
1. У каждого входного набора может не быть решений или может быть ровно одно решение.
2. Нельзя использовать один и тот же элемент (и один и тот же индекс) дважды.
3. Ответ можно возвращать в любом порядке нахождения индексов.

## Реализация алгоритма (`lab2.py`)
Для достижения максимальной производительности алгоритм реализован с использованием хэш-таблицы (словаря Python `dict`). Это позволяет находить пару чисел за один проход по массиву.

```python
def two_sum(nums: list[int], target: int) -> list[int]:
    """
    Находит индексы двух чисел, которые в сумме дают target.
    """
    seen = {}  # Словарь для хранения просмотренных чисел и их индексов {число: индекс}
    
    for current_index, num in enumerate(nums):
        complement = target - num  # Число, которое мы ищем для пары
        
        # Если дополнение уже встречалось ранее, возвращаем результат
        if complement in seen:
            return [seen[complement], current_index]
        
        # Сохраняем текущее число и его индекс в словарь
        seen[num] = current_index
        
    return []  # Возвращаем пустой список, если решение не найдено
    
    Оценка сложности алгоритма:
    Временная сложность (Time Complexity): $O(n)$, где $n$ — количество элементов в массиве nums. Мы обходим массив ровно один раз, а поиск и вставка элементов в хэш-таблицу (словарь) в среднем выполняются за амортизированное время $O(1)$.Пространственная сложность (Space Complexity): $O(n)$ в худшем случае, так как нам требуется память под словарь seen для хранения до $n$ элементов, если решение находится в самом конце массива или отсутствует.Тестирование (test.py)Для проверки корректности работы алгоритма написаны тест-кейсы, покрывающие базовые примеры из задания, а также случай отсутствия валидной пары.Pythonimport unittest
